**Платформа:** WordPress + Bricks Builder + ACF Pro + RankMath Pro

## Раздел 1: Архитектурные риски

### 1. Переусложнение

**Описание:** 6 CPT (Kitchen, Project, Review, Post, Page, FAQ), 3 таксономии (стиль, материал фасада, тип планировки), десятки ACF-полей на каждую модель. Заказчик не технический — при самостоятельном наполнении каталога (42 модели) высока вероятность пропуска полей, ошибок в заполнении, пустых карточек. Со временем часть полей перестаёт обновляться, структура деградирует.

- **Вероятность:** 4/5
- **Влияние:** 4/5
- **Митигация:**
  - Сократить обязательные поля CPT Kitchen до минимума: название, цена, тип, 1 фото. Остальные — опциональные.
  - Создать шаблоны заполнения с примерами (скриншоты + текстовая инструкция).
  - Настроить placeholder-значения и дефолтные изображения для незаполненных полей, чтобы карточка выглядела корректно даже с минимальными данными.
  - Валидация: обязательные поля помечены звёздочкой, ACF-группы с подсказками.

---

### 2. Мусорная структура

**Описание:** На текущем mebelit.site — 4 мёртвые страницы (404), дубли каталогов, пустой юридический текст в футере. При миграции на WordPress с 6 CPT и 3 таксономиями легко получить: пустые архивные страницы (например, `/stil-kuhni/loft/` с 0 кухонь), дубли контента между архивами и каталогами, тонкие страницы без содержания, которые размывают SEO.

- **Вероятность:** 3/5
- **Влияние:** 4/5
- **Митигация:**
  - Запускать таксономические архивы только когда в каждом термине >= 3 записей.
  - Настроить RankMath: `noindex` для архивов с < 3 записями.
  - Аудит структуры перед запуском: каждая страница должна иметь минимум 300 слов уникального контента или быть `noindex`.
  - Канонические URL строго проверить: один canonical на одну единицу контента.
  - Чеклист: проверить все 18 страниц на наличие контента перед переключением DNS.

---

### 3. SEO-риски

**Описание:** Переход с mebelit.site (0 органики, 0 индексации в Google) на новый домен кухнирема.рф. Риски: каннибализация между каталожными страницами и архивами CPT (например, `/uglovye-kuhni/` vs `/kuhnya/?type=uglovaya`), неправильные canonical при нескольких путях к одной модели, медленная индексация кириллического домена в Google, потеря тех минимальных позиций Яндекса, что есть.

- **Вероятность:** 3/5
- **Влияние:** 5/5
- **Митигация:**
  - Карта canonical: документ с указанием canonical для каждого URL-паттерна.
  - 301 редиректы: mebelit.site/straight → кухнирема.рф/pryamye-kuhni/ и т.д.
  - Google Search Console + Яндекс.Вебмастер подключить до запуска.
  - XML Sitemap подать вручную в обе поисковые системы.
  - Каталожные страницы (`/pryamye-kuhni/`) — canonical на себя, архивы CPT (`/kuhnya/`) — `noindex` или canonical на каталог.
  - Мониторинг индексации еженедельно первые 2 месяца.

### 4. Скорость

**Описание:** 205 фото кухонь (1080px), каждая модель — 3-5 фото. На каталожных страницах может отображаться 15-42 карточки с превью. Bricks Builder генерирует чистый CSS/HTML, но при неправильной настройке (глобальные стили, кастомный CSS, сторонние плагины) вес страницы растёт. Нет CDN на текущем VPS. Целевая аудитория — 76.7% мобильные (3G/4G в Красноярске, средняя скорость ниже Москвы).

- **Вероятность:** 4/5
- **Влияние:** 4/5
- **Митигация:**
  - WebP конвертация всех 205 фото + srcset (320/640/1080px).
  - Lazy loading для всех изображений ниже первого экрана.
  - Каталог: пагинация по 12 карточек или infinite scroll с подгрузкой.
  - Cloudflare CDN — бесплатный тариф достаточен.
  - Бюджет плагинов: максимум 8-10 активных (Bricks, ACF, RankMath, кэш, формы, безопасность + 2-3 утилитарных).
  - Кэширование: WP Super Cache или LiteSpeed Cache, критический CSS inline.
  - Цель: PageSpeed Insights >= 90 mobile, LCP < 2.5s, CLS < 0.1.

### 5. Управляемость

**Описание:** Заказчик — малый бизнес, не технический пользователь. Bricks Builder мощнее Tilda, но значительно сложнее. ACF-поля требуют понимания структуры. Обновления WordPress/Bricks/ACF могут сломать шаблоны. Нет документации для заказчика. При уходе разработчика заказчик остаётся с системой, которую не может поддерживать.

- **Вероятность:** 5/5
- **Влияние:** 4/5
- **Митигация:**
  - Разделить интерфейс: заказчик работает ТОЛЬКО через записи CPT (стандартный редактор WP + ACF-поля). Шаблоны Bricks — зона разработчика.
  - Создать видеоинструкцию (10-15 мин): как добавить/редактировать кухню, как изменить цену, как добавить фото.
  - Написать текстовую шпаргалку на 1 страницу (PDF).
  - Настроить роль «Менеджер контента» в WP — скрыть меню Bricks, плагинов, настроек.
  - Автообновления WordPress: выключить для мажорных версий, включить для патчей безопасности.
  - Bricks: зафиксировать версию, обновлять вручную после проверки на staging.

### 6. Мобильная версия

**Описание:** 76.7% трафика — смартфоны (5 589 из 7 287 визитов). Текущий mebelit.site использует костыль — дублирование Hero-секции для мобильного и десктопа вместо адаптивного дизайна. При переходе на Bricks риск повторить ошибку: desktop-first вёрстка, большие фото без srcset, квиз неудобен на маленьком экране, sticky-элементы (хедер, кнопка «Позвонить») перекрывают контент.

- **Вероятность:** 3/5
- **Влияние:** 5/5
- **Митигация:**
  - Строго mobile-first: начинать вёрстку каждого шаблона с viewport 375px.
  - Тестировать на реальных устройствах (не только DevTools): Android 10+, iOS 15+.
  - Квиз: кнопки вместо dropdown, шаги один за другим (не вкладки), крупные touch-target (минимум 48x48px).
  - Sticky-хедер: высота не более 56px, скрывать при скролле вниз, показывать при скролле вверх.
  - Sticky CTA («Рассчитать стоимость»): фиксированная кнопка внизу экрана, высота 48px, не перекрывает контент (padding-bottom на body).
  - Фото: `max-width: 100%`, srcset, aspect-ratio для предотвращения CLS.

### 7. "Красиво, но не продаёт"

**Описание:** Текущий mebelit.site: ~10 CTA-точек, но глубина просмотра — 1.37 страницы, среднее время — 1:23. Портфолио без призыва к действию. При новом дизайне риск увлечься анимациями, слайдерами, параллаксами — за счёт конверсии. Заказчик может просить «сделать красивее», что противоречит «сделать продающее».

- **Вероятность:** 3/5
- **Влияние:** 5/5
- **Митигация:**
  - Правило: каждый экран (viewport height) должен содержать или контент, или CTA. Не допускать «мёртвых зон» длиннее одного экрана.
  - Портфолио: каждый проект завершается кнопкой «Хочу такую кухню» → квиз.
  - Каталог: кнопка «Рассчитать стоимость» в каждой карточке.
  - Главная: квиз-превью в первые 2 экрана прокрутки.
  - Анимации: только функциональные (появление карточек при скролле). Никаких параллаксов, marquee, бесконечных каруселей.
  - Сократить CTA с ~10 до 4: квиз, замер, загрузка проекта конкурента, звонок.
  - Целевые метрики: глубина > 2.5, время > 2:30, конверсия > 3%.

### 8. Интеграции

**Описание:** Сайт интегрируется с n8n (webhook для форм/квиза), BlueSales API (CRM), Telegram (уведомления менеджеру), Яндекс Метрика. n8n webhook может быть недоступен при рестарте VPS. BlueSales API имеет лимит 500 записей и конфликт при одновременном доступе нескольких пользователей. Telegram уведомления зависят от стабильности бота.

- **Вероятность:** 3/5
- **Влияние:** 3/5
- **Митигация:**
  - Формы: fallback на email при недоступности webhook (wp_mail как резервный канал).
  - n8n: watchdog уже настроен (cron каждые 5 мин), P0-08 мониторит ошибки.
  - BlueSales: синхронизация через P0-05 каждый час, повторные попытки при ошибке «другой пользователь онлайн».
  - Telegram: проверка доступности бота в healthcheck (уже реализовано).
  - Яндекс Метрика: перенести счётчик 103970425, проверить цели после запуска.
  - Документировать все webhook URL и API-ключи в защищённом хранилище.

### 2.3 Вопросы заказчику

Эти вопросы необходимо задать до начала вёрстки. Без ответов работа будет содержать заглушки.

1. **Часы работы:** Какой график работы? Пн-Пт / Пн-Сб / Пн-Вс? С какого по какой час? Есть ли перерыв?
2. **Второй телефон:** Номер +7 (391) 292-77-04 из 2ГИС — актуален? Нужно ли его указывать на сайте?
3. **Рассрочка:** Актуальны ли условия «предоплата 10%, остальное после установки, рассрочка до 3 мес без банка»? Изменились ли условия?
4. **Линейки кухонь:** Все 42 модели из VK Market актуальны? Есть ли снятые с производства? Есть ли новые, которых нет в VK?
5. **Фото производства:** Можно ли организовать дополнительную фотосъёмку цеха? 3 текущих фото с сайта недостаточно для страницы «О компании».
6. **Фото команды:** Готовы ли к профессиональной фотосессии дизайнеров? Текущие фото непригодны (низкое качество).
7. **Реальные отзывы:** Есть ли клиенты, готовые оставить отзыв с фото установленной кухни? Разрешение на публикацию фото из VK Market?
8. **WhatsApp/Viber:** Какой мессенджер приоритетный для связи с клиентами? Актуальны ли все 4 канала (WhatsApp, Viber, SMS, Звонок)?
9. **Логотип:** Есть ли предпочтения по стилю логотипа «Кухни Рема»? Цветовые предпочтения? Или полностью доверяете дизайнеру?
10. **Домен кухнирема.рф:** Зарегистрирован ли домен? Если нет — нужно зарегистрировать до начала работ.

### 2.5 Ограничения для этапа 3

Что НЕЛЬЗЯ делать на этапе сборки сайта:

1. **Нельзя использовать Elementor, Divi, WPBakery** — только Bricks Builder. Решение зафиксировано в 01-stack-comparison.md.
2. **Нельзя создавать страницы без определённой роли** — каждая страница из 02-site-architecture.md имеет роль (конверсия, доверие, SEO, юридическая). Новые страницы только с обоснованием.
3. **Нельзя использовать 3D-рендеры как основной фотоконтент** — только реальные фото из VK Market. Рендеры допустимы как дополнение с пометкой «3D-визуализация».
4. **Нельзя добавлять больше 10 активных плагинов** — бюджет: Bricks, ACF Pro, RankMath Pro, кэш, безопасность, формы + 2-3 утилитарных. Каждый новый плагин = обоснование.
5. **Нельзя вёрстать desktop-first** — строго mobile-first (375px → 768px → 1024px → 1440px). 76.7% трафика — смартфоны.
6. **Нельзя создавать больше 4 CTA-точек на страницу** — квиз, замер, загрузка проекта, звонок. Никаких попапов, exit-intent, агрессивных баннеров.
7. **Нельзя публиковать фейковые отзывы** — только реальные (Flamp, 2ГИС, от клиентов с разрешением). 30 фейковых отзывов с mebelit.site удалены навсегда.
8. **Нельзя давать заказчику доступ к редактору Bricks** — только записи CPT через стандартный WP-редактор + ACF-поля. Роль «Менеджер контента».
9. **Нельзя запускать таксономические архивы с < 3 записями** — пустые архивы = тонкий контент = SEO-вред. Архив открывается только когда в нём >= 3 кухни.
10. **Нельзя подключать внешние скрипты без оценки влияния на скорость** — каждый внешний ресурс (чат, аналитика, шрифты) проверять через PageSpeed перед деплоем.
