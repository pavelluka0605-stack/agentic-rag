## 1. Дизайн-система

### 1.1 Цветовая система

Палитра строится на тёплых природных тонах (терракот, лесной зелёный, древесные оттенки) как эволюция тёмно-золотого визуала MEBELIT.site. Золотой (#e8c547) заменён на терракот (#C2613A) для большей теплоты и уникальности. Тёмный фон (#1a1a2e) сохраняется в ДНК-секциях (Hero, Footer, CTA-блоки).

#### Primary (основной бренд)

| Вариант | HEX | Роль | Где используется |
|---------|-----|------|-----------------|
| Primary | `#C2613A` | Основной акцент бренда | CTA-кнопки, ссылки, активные состояния, логотип |
| Primary Dark | `#9E4A2A` | Затемнённый акцент | Hover-состояния кнопок, активные элементы, подчёркивания |
| Primary Light | `#E8956F` | Мягкий акцент | Бейджи, теги, фоновые подложки, мягкие выделения |

#### Secondary (вспомогательный)

| Вариант | HEX | Роль | Где используется |
|---------|-----|------|-----------------|
| Secondary | `#3D6B5E` | Вторичный бренд-цвет | Вторичные кнопки, иконки преимуществ, секционные фоны |
| Secondary Dark | `#2C4F45` | Глубокий зелёный | Hover вторичных кнопок, футер (альтернатива тёмному) |
| Secondary Light | `#5A9484` | Светлый зелёный | Декоративные элементы, разделители, прогресс-бар квиза |

#### Accent (CTA-кнопки)

| Вариант | HEX | Роль | Где используется |
|---------|-----|------|-----------------|

| Accent Hover | `#A8522F` | Интерактивное состояние | Hover любого CTA-элемента |

#### Background (фоны)

#### Surface (карточки, модалки)

#### Text (текст)

| Название | HEX | Роль | Где используется |
|----------|-----|------|-----------------|

| Info | `#3B82A0` | Информация | Информационные подсказки, tooltip, уведомления |

#### Тёмная тема (ДНК MEBELIT)

Тёмная тема не является полноценной альтернативной темой. Она применяется точечно к конкретным секциям, сохраняя визуальную ДНК MEBELIT.site.

| Элемент | Значение | Где |
|---------|----------|-----|
| Фон тёмных секций | `#1B1B2F` (эволюция #1a1a2e) | Hero-баннер, Footer, CTA-блоки между секциями |
| Текст на тёмном | `#FFFFFF` / `rgba(255,255,255,0.85)` | Заголовки / обычный текст на тёмных секциях |
| Акцент на тёмном | `#C2613A` (терракот вместо золотого) | CTA-кнопки и ссылки на тёмном фоне |
| Вторичный на тёмном | `#E8956F` (Primary Light) | Мелкие акценты, иконки на тёмном фоне |
| Border на тёмном | `rgba(255,255,255,0.12)` | Разделители в футере, линии |

**Правило:** Тёмные секции используются максимум 3 раза на странице — Hero, один CTA-блок, Footer. Между двумя тёмными секциями всегда минимум 2 светлые.

### 1.7 Блоки преимуществ

Эволюция блока «Что выделяет нас» и «Преимущества производства» с MEBELIT.site — визуально чище, структурированнее.

#### На светлом фоне

```
Секция:
  Background: var(--color-bg) или var(--color-bg-alt)
  Padding: 80px 0 (десктоп) / 48px 0 (мобильная)

Сетка:
  Десктоп: 4 карточки в ряд (grid-template-columns: repeat(4, 1fr), gap 24px)
```

## 2. UX-принципы

### 2.1 Путь пользователя

#### Основной сценарий (рекламный трафик — 90% визитов)

```
Яндекс Директ / VK Реклама
  → Hero (3 секунды на решение: остаться или уйти)
    → Сканирование: заголовок + фото кухни + цена + CTA
      → Вариант A: CTA → квиз → заявка (быстрый путь)
      → Вариант B: Скролл → каталог → карточка → квиз → заявка
      → Вариант C: Скролл → отзывы/портфолио → доверие → CTA → заявка
```

#### Вторичный сценарий (органика, повторные визиты)

```
Поиск «кухни на заказ Красноярск»
  → Главная или каталожная страница (SEO-посадочная)
    → Навигация по каталогу (фильтры по типу/стилю/цене)
      → Детальная страница кухни → «Рассчитать по моим размерам» → квиз
    → О компании → портфолио → «Хочу так же» → форма
```

#### Ключевые метрики для отслеживания

| Этап | Метрика | Целевое значение |
|------|---------|-----------------|
| Hero → скролл | Scroll depth > 30% | > 60% |
| Hero → CTA | CTR кнопки «Рассчитать» | > 5% |
| Каталог → карточка | CTR карточки | > 15% |
| Квиз старт → заявка | Completion rate | > 40% |
| Общая конверсия | Визит → заявка | > 3% |

---

### 2.2 Принцип «Быстрое действие»

Быстрое действие = пользователь не думает, а действует импульсивно. Применяется там, где барьер принятия решения низкий.

| Где | Элемент | Действие | Почему быстрое |
|----|---------|----------|----------------|
| Hero | CTA-кнопка «Рассчитать стоимость» | Открывает квиз | Бесплатно, без обязательств, любопытство |
| Sticky CTA (мобильная) | Нижняя полоса | Звонок или квиз | Всегда видима, один тап |
| Click-to-call | Телефон в header | Звонок | Для «горячих» клиентов, один тап на мобильной |
| Каждая карточка | «Рассчитать по моим размерам» | Квиз с предзаполненной моделью | Привязка к конкретной кухне |

**Правило:** Текст быстрых CTA не должен содержать слова «купить», «заказать», «оплатить». Используем: «Рассчитать», «Узнать стоимость», «Получить расчёт» — низкий барьер входа.

---

### 2.3 Принцип «Дожатие доверием»

Доверие перед действием = пользователь сначала убеждается, потом действует осознанно. Применяется там, где нужно преодолеть скептицизм.

| Сценарий | Последовательность |
|----------|-------------------|
| После каталога | Каталог кухонь → Реальные отзывы (Flamp/2ГИС) → CTA «Получить расчёт» |
| Портфолио | Фото проекта → Детали (размеры, материалы, срок) → «Хочу такую же» |
| О компании | Команда с фото → Производство → Гарантии → «Пригласить дизайнера» |
| Квиз | Шаг 1-3 (вовлечение) → «Ваш проект подготовят Павел и Николай» → Шаг 4 (контакты) |

**Правило:** Между контентной секцией (каталог, портфолио) и CTA-блоком всегда стоит секция доверия (отзывы, преимущества, команда).

---

### 2.4 Принцип «Меньше текста»

Где текст = враг вовлечения. Пользователь сканирует, не читает.

| Секция | Максимум текста | Что вместо |
|--------|----------------|------------|
| Hero | Заголовок (до 8 слов) + подзаголовок (до 15 слов) + CTA | Крупное фото кухни говорит само |
| Карточка каталога | Название + цена + 1 строка мета | Фото продаёт, текст информирует |
| CTA-блок | 1 предложение + кнопка | Эмоция, не аргументация |
| Преимущества | Заголовок (3-4 слова) + 1 предложение | Иконка передаёт суть быстрее |
| Бейджи | 1-2 слова | Визуальный маркер, не описание |

---

### 2.5 Принцип «Больше текста»

Где текст = инструмент убеждения и SEO. Пользователь пришёл за информацией.

| Секция | Объём текста | Цель |
|--------|-------------|------|
| О компании | 500-800 слов | Доверие, история, ценности, SEO |
| Детальная страница кухни | 300-500 слов | Материалы, фурнитура, характеристики, SEO |
| Блог / SEO-статьи | 1000-2000 слов | Органический трафик, экспертность |
| FAQ | 15-20 вопросов-ответов | Снятие возражений, SEO (structured data) |
| Описание процесса работы | 200-400 слов + пошаговая визуализация | Прозрачность, снятие страха «а как это будет?» |

---

### 2.6 Как не перегрузить главную

На MEBELIT.site было 15 секций, включая дубли и мусор. На кухнирема.рф — максимум 10-12 секций, каждая с одной задачей.

#### Правила

1. **Максимум 10-12 секций** на главной странице
2. **Каждая секция решает одну задачу:** продаёт, доказывает, информирует или конвертирует
3. **Нет двух CTA-блоков подряд** — между ними всегда контентная секция
4. **Чередование:** продающий блок (каталог) → доверительный блок (отзывы) → конверсионный блок (CTA)
5. **Нет двух тёмных секций подряд** — между ними минимум 2 светлые

#### Рекомендуемая структура главной

| # | Секция | Тип | Фон | Задача |
|---|--------|-----|-----|--------|
| 1 | Hero | Продающий | Тёмный (#1B1B2F) | Зацепить за 3 сек, направить в квиз |
| 2 | Преимущества (3-4 шт.) | Доверие | Светлый | Почему мы: шаг 1 см, от 21 дня, гарантия |
| 3 | Каталог-превью (3 типа) | Продающий | Альтернативный (#F5EDE3) | Показать ассортимент, цены |
| 4 | CTA-блок | Конверсия | Тёмный | «Узнайте точную стоимость за 5 минут» |
| 5 | Портфолио (4-6 проектов) | Доверие | Светлый | Реальные фото → доказательство |
| 6 | Процесс работы (5 шагов) | Информирование | Альтернативный | Снять страх, показать прозрачность |
| 7 | Отзывы (карусель) | Доверие | Светлый | Реальные отзывы из Flamp/2ГИС |
| 8 | Рассрочка + оффер | Продающий | Альтернативный | Снять ценовой барьер |

### 2.7 Формы: удобство без дешевизны

Формы на MEBELIT.site работали (квиз — лучший элемент), но их было слишком много (10+). На кухнирема.рф — 4 стратегических формы.

#### 4 формы на сайте

| Форма | Где | Поля | CTA-текст |
|-------|-----|------|-----------|
| Квиз (4 шага) | Встроен на главной + отдельная /quiz | Планировка → размеры → материал → контакты | «Получить расчёт» |
| Быстрая заявка | Модалка (из карточек каталога) | Имя + телефон + модель (предзаполнено) | «Рассчитать стоимость» |
| Замер | Модалка (из header CTA и CTA-блоков) | Имя + телефон + адрес (опционально) | «Записаться на замер» |
| Загрузка проекта | Модалка (механика «пришлите проект конкурента») | Имя + телефон + файл | «Получить выгодное предложение» |

#### UX-правила форм

| Правило | Реализация |
|---------|-----------|
| Крупные поля | Min-height 48px, font-size 16px (предотвращает zoom на iOS) |
| Видимые лейблы | Label всегда над полем, не placeholder-only |
| Прогресс-бар (квиз) | Визуальная полоса вверху: `var(--color-primary)`, высота 4px |
| Визуальные иконки (квиз) | Большие карточки выбора с иконками для планировок и способов связи |
| Текст результата на кнопке | «Получить расчёт» (не «Отправить», не «Далее»), крупная, акцентная |
| Обратная связь | После отправки: анимация success + «Спасибо, [Имя]! Мы свяжемся с вами в течение 15 минут» |
| Способы связи | Выбор: WhatsApp / Telegram / Звонок (сохраняем от MEBELIT) |

### 2.8 Мобильный UX

77% трафика MEBELIT.site — мобильные устройства. Mobile-first — не опция, а требование.

| Принцип | Реализация |
|---------|-----------|
| Touch-targets | Все интерактивные элементы min 44x44px |
| Горизонтальный скролл | Карточки каталога и отзывов — swipe-карусель (не вертикальный список) |
| Sticky нижняя полоса | `[Позвонить]` + `[Рассчитать стоимость]` — фиксирована, появляется после скролла Hero |
| Бургер-меню | Полноэкранный overlay (slide-in from right), не dropdown |
| Фото | Swipe-галерея (touch-gesture), не pinch-to-zoom |
| Формы | Input type="tel" для телефона, autocomplete, min-height 48px |
| Шрифты | min 14px для любого читаемого текста (WCAG) |
| Отступы | Уменьшены по шкале: section padding 48px → 48px, внутренние gap 24px → 16px |
| Кнопки | Полная ширина контейнера для Primary CTA |
| Таблицы | Горизонтальный скролл или адаптация в карточки |

### 2.9 Улучшения UX vs MEBELIT.site

| Что было (MEBELIT) | Что стало (Кухни Рема) | Зачем |
|---------------------|------------------------|-------|
| Двойной Hero (десктоп + мобильная копия) | Один адаптивный Hero (responsive) | Убираем костыль, чистый код, нет дублирования контента |
| ~10 CTA-точек + 5 форм + попапы | 4 стратегические формы + 3 CTA-точки | Снижаем давление, повышаем качество каждого касания |
| Фейковые отзывы (все 5 звёзд, дубли) | Реальные отзывы из 2ГИС/Flamp (со скриншотами) | Доверие. Смешанные отзывы (4-5 звёзд) убедительнее идеальных |
| Только 3D-рендеры кухонь | Реальные фото из VK (205 шт.) + рендеры с пометкой | Честность. Бейдж «Реальное фото» / «3D-визуализация» |
| Нет детальных страниц кухонь | CPT Kitchen: галерея, характеристики, материалы, цена | Вовлечение. Заинтересованный клиент может углубиться |
| Мусорный англоязычный контент | Удалён полностью | Доверие. Профессионализм |
| Tilda (шаблонные блоки, медленный SEO) | WordPress + Bricks (class-based CSS, семантичный HTML) | Скорость, SEO, гибкость, масштабируемость |
| Нет «О компании», портфолио, FAQ | Полноценные страницы с контентом | SEO + доверие + вовлечение |
| Jivo-чат (стандартный) | Claude AI чат (chat.mebelit.ru) — уникальный | Конкурентное преимущество, автоматизация |
| Золотой (#e8c547) + тёмный (#1a1a2e) — холодно | Терракот (#C2613A) + тёплый белый (#FDFAF6) — уютно | Эмоция «тепло, дом, уют» вместо «дёшево, тёмно» |
| Open Sans (текст) | Inter (текст) | Лучшая экранная читаемость, современнее |
| Нет SEO (0 органических визитов) | RankMath + Schema.org + блог + geo-targeted контент | Снижение зависимости от рекламного бюджета |
| Позиционирование «дёшево до 50%» | «Индивидуальность по цене серийной» (УТП шаг 1 см) | Бренд вместо дискаунта, лояльные клиенты вместо ценовых охотников |
